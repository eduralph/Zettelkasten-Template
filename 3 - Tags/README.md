# 3 - Tags

**Tag glossary.** One note per tag, explaining what the tag means and when to use it.

## Purpose

Tags drift. Without a glossary, `#philosophy` ends up meaning slightly different things over time, and similar-but-distinct concepts get conflated. This folder is the source of truth for what each tag *officially* means.

## Conventions

- **Filename:** the tag itself, without the `#` (e.g., `philosophy.md`, `mortality.md`).
- **Body:** a definition, scope (when to use it / when *not* to use it), and optionally a [[0 - Meta/Garden|Dataview]] block listing notes that carry the tag.
- **Hierarchy:** if you use nested tags (`#philosophy/stoicism`), create a note per level or document the hierarchy in the parent tag's note.

## When to Add a Tag Note

- The first time you use a tag, ask: "will I use this again, or is this a one-off?"
- If recurring, create the glossary entry here immediately. Cheap to do early, expensive to do later.
