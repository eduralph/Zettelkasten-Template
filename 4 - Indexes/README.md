# 4 - Indexes

**Maps of Content (MOCs).** Hand-curated entry points that link related notes in narrative order — table-of-contents views of your thinking.

## Purpose

Tags and the graph view show *what* is related. Indexes explain *how* and *why* — and in what order to read.

An index is the most opinionated kind of note in the vault: it implies a structure, an argument, or a story arc across the notes it gathers.

## When to Create an Index

- When 5+ evergreen notes in [[6 - Zettelkasten/About|6 - Zettelkasten]] cluster around a theme.
- When you find yourself navigating between the same set of notes repeatedly.
- When you're starting to draft a chapter, essay, or talk — the index is the outline.

## Conventions

- **Filename:** descriptive theme name (e.g., `Mortality and Meaning.md`, `Probability and Fortune.md`).
- **Structure:** intro paragraph → ordered list of linked notes with one-line context for each → optional "see also" section linking to adjacent indexes.
- **Difference from Garden views**: indexes are *static* and *opinionated*. Dashboards in [[0 - Meta/README|0 - Meta]] are *dynamic* and *neutral*. Don't replace an index with a Dataview query — the value of an index is the human curation.
